# Control Work Generator

MVP-прототип программного обеспечения для автоматизации формирования контрольных работ и фондов оценочных средств по рабочим программам дисциплин с применением NLP-подходов и локально развернутых языковых моделей.

Проект ориентирован на закрытое развертывание в контуре образовательной организации: рабочие программы дисциплин и оценочные материалы не требуется передавать во внешние публичные сервисы.

## Возможности MVP

- загрузка РПД в форматах DOCX, PDF, TXT;
- извлечение текста из документа;
- структурный анализ РПД: темы, компетенции, результаты обучения и разделы;
- создание структуры фонда оценочных средств;
- редактирование матрицы компетенций и индикаторов;
- локальный банк оценочных заданий;
- три режима генерации заданий: шаблонный, гибридный и Ollama;
- автоматическая проверка покрытия тем и компетенций;
- выявление потенциальных дублей;
- экспорт контрольной работы и полного ФОС в DOCX;
- REST API на FastAPI;
- frontend-интерфейс на React + Vite.

## Подключение локальной LLM через Ollama

Локальная LLM необязательна: приложение продолжает работать в шаблонном режиме, если Ollama не установлена или временно недоступна.

### 1. Установить Ollama

Для Windows можно выполнить команду в PowerShell:

```powershell
irm https://ollama.com/install.ps1 | iex
```

Либо скачать установщик с официального сайта Ollama.

### 2. Скачать локальную модель

Для первого запуска рекомендуется начать с модели `qwen2.5:7b`:

```powershell
ollama pull qwen2.5:7b
```

Для менее производительного компьютера можно использовать более компактную модель:

```powershell
ollama pull qwen2.5:3b
```

### 3. Создать локальный `.env`

В папке `backend` скопировать пример:

```powershell
Copy-Item .env.example .env
```

Содержимое по умолчанию:

```text
OLLAMA_BASE_URL=http://127.0.0.1:11434
OLLAMA_MODEL=qwen2.5:7b
OLLAMA_GENERATION_TIMEOUT_SECONDS=180
OLLAMA_ALLOW_PUBLIC_ENDPOINT=false
APP_SECRET_KEY=change-me-in-production
```

Для реального развертывания замените `APP_SECRET_KEY` на собственное случайное значение.

### 4. Проверить подключение

После запуска backend откройте проект ФОС и нажмите кнопку:

```text
Проверить Ollama
```

Если локальный сервер доступен, интерфейс покажет список установленных моделей.

## Режимы формирования банка заданий

### Шаблонный режим

Работает полностью локально без языковой модели. Формирует предсказуемые заготовки, которые преподаватель может отредактировать вручную.

### Гибридный режим

Сначала формирует структуру банка по правилам, затем улучшает ограниченное количество заданий через локальную LLM. Остальные элементы остаются шаблонными заготовками. Подходит для первого запуска и для компьютеров с ограниченными ресурсами.

### Режим Ollama

Передает все формируемые задания локальной модели. Для каждого задания модель возвращает структурированный JSON:

```json
{
  "text": "формулировка задания",
  "answer": "эталонный ответ",
  "criteria": ["критерий 1", "критерий 2", "критерий 3"]
}
```

При включенном резервном режиме ошибки Ollama не останавливают процесс: проблемные элементы сохраняются как шаблонные заготовки.

## Основной сценарий работы

1. Зарегистрироваться и войти в систему.
2. Загрузить РПД.
3. Проверить выделенные темы, компетенции и результаты обучения.
4. Создать проект ФОС.
5. При необходимости исправить матрицу компетенций и индикаторов.
6. Выбрать режим генерации банка заданий.
7. Сформировать задания.
8. Проверить покрытие тем, компетенций и наличие дублей.
9. Отредактировать проблемные элементы.
10. Экспортировать полный ФОС в DOCX.

## Исследовательская идея

В системе применяется гибридный подход: структурный анализ РПД, правила, шаблоны и локально развернутая языковая модель. РПД и оценочные материалы остаются внутри закрытого контура образовательной организации. На следующих этапах локальный интеллектуальный модуль может быть дополнен RAG-контуром и дообучением на реальных РПД и соответствующих фондах оценочных средств.
