# Control Work Generator

MVP-прототип программного обеспечения для автоматизации формирования контрольных работ и фондов оценочных средств по рабочим программам дисциплин с применением структурного анализа РПД, шаблонной генерации и узкоспециализированного генератора ФОС на экспертных примерах.

Проект ориентирован на закрытое развертывание в контуре образовательной организации: рабочие программы дисциплин и оценочные материалы не требуется передавать во внешние публичные сервисы.

## Возможности MVP

- загрузка РПД в форматах DOCX, PDF, TXT;
- извлечение текста из документа;
- структурный анализ РПД: темы, компетенции, результаты обучения и разделы;
- создание структуры фонда оценочных средств;
- редактирование матрицы компетенций и индикаторов;
- локальный банк оценочных заданий;
- режимы генерации заданий: шаблонный, по экспертным примерам, узкая модель ФОС и гибридный режим;
- накопление обучающей выборки через экспертную разметку заданий;
- экспорт обучающей выборки в JSONL;
- автоматическая проверка покрытия тем и компетенций;
- выявление потенциальных дублей;
- экспорт контрольной работы и полного ФОС в DOCX;
- REST API на FastAPI;
- frontend-интерфейс на React + Vite.

## Узкоспециализированный генератор ФОС

В проекте больше не используется идея внешней универсальной LLM как ядра генерации. Вместо этого целевая архитектура строится вокруг узкоспециализированного генератора ФОС, который работает с данными самой системы:

- темами из РПД;
- компетенциями и индикаторами;
- типами оценочных материалов;
- уровнем сложности;
- экспертно подтвержденными примерами;
- плохими примерами и примерами на доработку.

Система поддерживает накопление обучающей выборки: преподаватель редактирует задание и сохраняет его как хороший пример, плохой пример или пример, требующий доработки. Эти данные затем экспортируются в JSONL и могут использоваться для обучения узкой модели.

Пример настроек:

```text
NARROW_LLM_MODEL_PATH=storage/models/narrow_fos_model.json
NARROW_LLM_MODEL_VERSION=narrow-fos-generator-v0.1
NARROW_LLM_MAX_ITEMS=40
NARROW_LLM_FALLBACK_TO_TEMPLATE=true
APP_SECRET_KEY=change-me-in-production
```

## Режимы формирования банка заданий

### Шаблонный режим

Работает полностью локально. Формирует предсказуемые заготовки, которые преподаватель может отредактировать вручную.

### Режим по экспертным примерам

Использует хорошие задания из обучающей выборки как образцы и адаптирует их под новые темы, компетенции и типы контроля.

### Узкая модель ФОС

Использует экспертно подтвержденные примеры как специализированную обучающую основу для формирования формулировки задания, эталонного ответа и критериев оценивания.

### Гибридный режим

Комбинирует узкую модель ФОС и шаблонный генератор. Если обучающих примеров недостаточно, система может использовать резервный шаблонный режим.

## Основной сценарий работы

1. Зарегистрироваться и войти в систему.
2. Загрузить РПД.
3. Проверить выделенные темы, компетенции и результаты обучения.
4. Создать проект ФОС.
5. При необходимости исправить матрицу компетенций и индикаторов.
6. Выбрать режим генерации банка заданий.
7. Сформировать задания.
8. Проверить покрытие тем, компетенций и наличие дублей.
9. Отредактировать проблемные элементы.
10. Сохранить удачные и неудачные задания в обучающую выборку.
11. Экспортировать обучающую выборку JSONL или полный ФОС в DOCX.

## Исследовательская идея

В системе применяется гибридный подход: структурный анализ РПД, правила, шаблонное формирование каркаса заданий, генерация по экспертным примерам и накопление обучающей выборки для узкоспециализированной модели ФОС. РПД и оценочные материалы остаются внутри закрытого контура образовательной организации.
